Pattern Flags

Regular expressions may contain flags that enable or disable certain behavior in the regular expression engine. They are enabled using the special syntax:

(?dimsux-dimsux)

Letters preceeded by a '-' will turn off the feature. The flags must appear before the pattern to have an affect. That is, the pattern "nocase(?i)" will not match NOCASE because the flag is at the end of the expression. By the same token, this can be used to some advantage. The expression "UPPER(?i)NOCASE(?-i)lower" will match either "UPPERnocaselower" or "UPPERNOCASElower" or any combination of upper and lower case characters in the "NOCASE" subsequence.

The flags and their meanings are as follows:

Flag Meaning
d unix lines; only the '\n' character is recognized as a line terminator in this mode.
i case insensitive; enables case-insensitive matching. This feature may impose a performance penalty.
m multi-line mode; in multiline mode the expressions ^ and $ match just after or just before, respectively, a line terminator or the end of the input sequence. By default these expressions only match at the beginning and end of the entire input sequence.
s single-line mode; the . (period) metacharacter will match line terminators. Normally it does not.
u Unicode case mode; when used with the i flag this performs the case-insensitive matching in a manner consistent with the Unicode Standard. By default, characters are assumed to be US-ASCII. This feature may impose a performance penalty.
x comments; in this mode, whitespace is ignored and embedded comments starting with # are ignored until the end of the line.

Some examples

Pattern Matches
(?i)nocase NOCASE, nocase, NOcase, NoCaSe ...
(?ix)pattern # comment pattern, PATTERN, PaTtErN ... (and ignores the comment)
